Why Dozenal?

🔢 The Decimal Problem

  • Base-10 has only factors 2 and 5
  • 1/3 = 0.333...   1/6 = 0.166...   (infinite!)
  • Real-world consequences: Pentium FP Bug (1994), Ariane 5 crash (1996)

✨ The Dozenal Advantage

  • Base-12 has factors 2, 3, 4, 6
  • 1/3 = 0.4   1/4 = 0.3   1/6 = 0.2
  • Already used: 12 hours, 12 months, dozens, 12 inches/ft

🎯 Our Approach

Challenge: How to encourage the next generation to explore dozenal counting?
Solution: Game-Based Learning — A multiplayer card game (based on Crazy Eights) where players naturally practice base-12 arithmetic through gameplay.

Crazy 1-0's 05 / 10

Game Rules Engine

🃏

Suit / Rank Match

Play a card matching the top card's suit or rank — classic Crazy Eights foundation.

➕

Sum Rule ⭐

Play a numeric card if your card + top card = target.
Dozenal: 5 + 7 = 10₁₂ (=12₁₀) ✓

🌟

Wildcard "1-0"

The 10 card is always playable. Player chooses a forced suit for the next player.

⏭️

Skip Card

Playing 6 (doz) or 5 (dec) skips the opponent — you get a free play!

📥

Draw Limit (3)

Max 3 draws per turn. Still no playable card? Auto-pass + opponent gets a free play.

🔢

Dual Number Systems

Switch between Decimal and Dozenal at game creation.
Symbols: 0–9, ↊ (10), ↋ (11)

Multi-Round Scoring

Winner earns points = total value of loser's remaining cards. First to the target score wins the match. All scores display in the chosen base (e.g. "1↊" = 22₁₀ in dozenal mode).

Crazy 1-0's 06 / 10

Arithmetic Challenge ⭐

How It Works

  • Play a face card (J, Q, K, C) → triggers a math challenge
  • J, Q → random operation (+, −, ×, ÷)
  • K (dec) / C (doz) → player chooses the operation
  • Both players see the question simultaneously
  • First to answer correctly wins bonus points!

Educational Core 🎓

  • Questions display in the current numeral system
  • In dozenal mode → dozenal numbers + dozenal numpad
  • Players practice base-12 arithmetic naturally
  • Competitive element boosts engagement

Server-Authoritative

Answer validation happens on the backend — prevents cheating, ensures fair "first responder" arbitration.

Additional Features

🔐

Authentication

JWT-based login/register + Guest Mode for quick access. Sessions persist across page reloads.

🚪

Lobby & Matchmaking

Create room → share ID → opponent joins. Choose dec/doz mode at creation. Empty rooms auto-clean.

💬

In-Game Chat

Collapsible glassmorphic sidebar with left/right toggle. Auto-scrolls on new messages.

👑

Admin Dashboard

Force-logout users, manage rooms, search accounts, reset passwords, clear match history.

🔄

Rematch System

Request/Accept/Decline rematch after game over. Mutual agreement auto-starts a new game.

📡

Disconnect Alerts

Clear notifications when opponent disconnects mid-game in order to prevent user confusion.

👤 Profile Page

View match history, win/loss statistics, and account settings. All data persisted in PostgreSQL.

Ubuntu EC2 instance with Docker installed

3. Deploy
docker compose up -d — one-command deploy
4. Run

Repository Pattern

Dev: In-Memory storage (fast iteration)
